# 02 — Database Schema

## Goal
All SQLAlchemy 2.x models and the initial Alembic migration exist; the app creates/updates its SQLite schema automatically on startup.

## Context
The schema is the ported abn-analyst model plus the new audit/sharing tables (spec: Architecture → Data model delta). Every later step depends on it.

## Prerequisites
01-project-setup.

## Tasks
1. Port models from `abn-analyst/app/database.py` into `core/models.py` using SQLAlchemy 2.x declarative style: `Transaction` (all columns incl. `description_structured`, `categorization_source`, `source_file/line`, `transaction_hash`, indexes), `CategorizationRule` (+ **new** `uuid` column, unique, default uuid4), `RuleCondition`, `Budget`. Drop users/auth models.
2. Add new tables: `rule_change_reports` (id, created_at, rule_id, rule_uuid, action enum: create/update/delete/toggle/recategorize/import, rule_before JSON, rule_after JSON, summary counts), `rule_change_items` (report_id FK cascade, transaction_id, old/new category, old/new tags), `download_state` (source, account, last_success_at, last_range_end), `snapshot_imports` (created_at, source machine id, schema_version, counts JSON, overwrites JSON).
3. Alembic setup (`alembic.ini`, env targeting the data-dir DB) with a single autogenerated initial revision; `alembic upgrade head` runs on app startup.
4. Session management: engine/session factory bound to the data-dir path; `get_db` dependency.
5. Tests first: model round-trips for each table, rule uuid auto-generation, cascade delete of conditions and change items, startup creates schema in a fresh temp data dir, second startup is a no-op.

## Acceptance Criteria
- Fresh data dir → startup produces a DB with all tables; `alembic current` shows head.
- All model unit tests green; `ruff check .` clean.

## Notes
- > ⚠ Golden Principle 6: schema changes via Alembic only — do not port the legacy `ensure_*` functions.
- Keep legacy column names (`accountNumber`, `transactiondate`, …) so the legacy migration (step 12) and snapshot format stay simple.
